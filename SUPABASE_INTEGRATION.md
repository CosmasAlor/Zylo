#### `lib/supabase/server.ts`
- Server-side Supabase client for use in Server Components
- Handles cookie-based session management
- Usage:
```typescript
import { createClient } from '@/lib/supabase/server'
import { cookies } from 'next/headers'

const cookieStore = await cookies()
const supabase = createClient(cookieStore)
const { data } = await supabase.from('table_name').select()
```

#### `lib/supabase/client.ts`
- Browser-side Supabase client for use in Client Components
- Use with `'use client'` directive
- Usage:
```typescript
'use client'
import { createClient } from '@/lib/supabase/client'

const supabase = createClient()
const { data } = await supabase.from('table_name').select()
```

#### `lib/supabase/middleware.ts`
- Middleware for automatically refreshing user sessions
- Keeps authentication tokens up-to-date
- Usage in middleware.ts or similar

## Next Steps

### 1. Update Database Strategy
Choose one approach:

**Option A: Keep Prisma (Recommended)**
- Continue using Prisma as your ORM for PostgreSQL
- Supabase is just a hosted PostgreSQL provider
- Use `DATABASE_URL` pointing to Supabase connection string
- Use Supabase client helpers only for authentication/real-time features

**Option B: Migrate to Supabase JavaScript Client**
- Replace Prisma entirely with Supabase client
- Suitable for simpler projects without complex queries
- Requires refactoring all data access code

### 2. For Local Development
Current setup uses local PostgreSQL:
- `DATABASE_URL` → Local database (Prisma)

## Using Supabase Features

### Real-Time Subscriptions
```typescript
'use client'
import { createClient } from '@/lib/supabase/client'

const supabase = createClient()

const channel = supabase
  .channel('table-changes')
  .on('postgres_changes', 
    { event: '*', schema: 'public', table: 'todos' },
    (payload) => console.log('Change:', payload)
  )
  .subscribe()

// Cleanup
channel.unsubscribe()
```

### Authentication (Optional)
```typescript
// Sign up
const { data, error } = await supabase.auth.signUp({
  email: 'user@example.com',
  password: 'password',
})

// Sign in
const { data, error } = await supabase.auth.signInWithPassword({
  email: 'user@example.com',
  password: 'password',
})

// Get session
const { data } = await supabase.auth.getSession()
```

## Troubleshooting

### "Cannot find module '@supabase/ssr'"
```bash
npm install @supabase/supabase-js @supabase/ssr
```

### "NEXT_PUBLIC_SUPABASE_URL is undefined"
Ensure variables are added to `.env.local` and restart dev server:
```bash
npm run dev
```

### Supabase connection issues
Check:
1. Supabase project status (Settings → General)
2. Connection string is correct
3. Network allows external connections
